Working on this game was my first time implementing enemy AI and while it definitely had a learning curve, I ended up really enjoying it and am super interested in doing AI work on future projects. I decided to utilize behavior trees to implement the AI because I wanted to learn how to program a frequently used system without choosing something too complicated. The main thing that I learned will bringing the enemies to life is that coding an enemy's attacks is actually the easy part, but trying to get them move to around their environment naturally is an adventure. To implement the movement I ended up using context steering because it seemed effective for creating realistic behavior in our game's large rooms with occasionaly obstacles and also allowed for heavy amounts of tweaking/customization based on how I wanted the enemy to react to certain things around it.
