Ensol

Enemy Creation Process

First, I would meet with our lead combat designer to write a design document for the enemy. This made sure we were on the same page for how the enemy would work and allowed me to begin brainstorming how I would implement them.
Then I would draft up a plan for the enemy's behavior tree. This allowed me to see how much previous enemy code I could reuse and what new behaviors/nodes would need to be written.
After that came creating a prototype of the enemy. This prototype would be tweaked heavily based on playtesting until we were satisfied with its behavior.
Lastly, I would implement the finalized model and animations. I would also often come back to add bits of polish to enemies as I learned new things when developing subsequent enemies.

Other Work I did on the Project

In order to playtest enemies and combat mechanics more easily, I worked with a group member to create a playtesting environment. In this mode, the user can enable/disable all the combat mechanics as well as spawn in enemies.
I programmed a lot of the on screen UI for the game including health and tutorial text. Adding little animations and polish to make the UI more interesting to look at was one of my favorite things to program.
